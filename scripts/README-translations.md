# Как добавить переводы для коллекций

## Проблема

После первого запуска скрипта `add-collection-translations.ts` некоторые коллекции уже получили русские переводы, но их `slug` теперь на русском языке. При повторном запуске скрипт ищет переводы по текущему `slug` коллекции, но не находит, потому что ключи в объекте `TRANSLATIONS` на английском.

## Решение

Скрипт был обновлен и теперь:
1. Ищет английский `slug` в массиве `translations`, если текущий `languageCode` не английский
2. Пробует найти перевод по английскому названию коллекции
3. Выводит информацию о том, какие ключи он пробовал

## Что делать, если коллекции все еще не переводятся

### Вариант 1: Добавить маппинг по русскому slug

Если коллекция уже имеет русский slug, добавь его в объект `TRANSLATIONS`:

```typescript
const TRANSLATIONS: Record<string, { name: string; slug: string }> = {
  // Английские ключи
  computers: { name: 'Компьютеры', slug: 'kompyutery' },
  equipment: { name: 'Оборудование', slug: 'oborudovanie' },
  
  // И русские ключи (на случай если slug уже изменен)
  kompyutery: { name: 'Компьютеры', slug: 'kompyutery' },
  oborudovanie: { name: 'Оборудование', slug: 'oborudovanie' },
};
```

### Вариант 2: Обновить скрипт для автоматического поиска

Скрипт уже обновлен и должен находить переводы автоматически. Если не работает, проверь логи - там будет указано, какие ключи скрипт пробовал.

## Структура объекта TRANSLATIONS

Ключ - это английский slug коллекции (например, `computers`, `equipment`).

Значение - объект с русским названием и русским slug:

```typescript
equipment: {
  name: 'Оборудование',  // Русское название для отображения
  slug: 'oborudovanie',  // Русский slug для URL
}
```

## Как проверить, какой slug у коллекции

В логах скрипта будет показано:
- Текущий `name` коллекции
- Текущий `slug` коллекции
- `languageCode` коллекции
- Какие ключи скрипт пробовал для поиска перевода

Используй эту информацию, чтобы добавить правильный ключ в `TRANSLATIONS`.
